## 2. ユーザー価値

- 対象ユーザー: 保存済み WordPack を繰り返し探し、開き、学習・公開・削除を管理するログイン済みユーザー
- 利用文脈: 右側の作成・生成キューを表示したまま、多数の WordPack をリスト表示で走査する場面
- ユーザー目的: 対象の見出し語、生成状態、公開状態、更新日、学習状況を誤認せず、必要な操作へ進む
- 支援するタスク: 一覧走査、プレビュー、例文生成、音声再生、語義確認、公開設定、削除
- このUIが助ける理解・判断・行動: 1件を1行として上から下へ読め、主操作「開く」と補助・危険操作の境界が分かる
- このUIがなければ困る点: 2列化と固定幅の操作群が主領域を超え、見出し語が幅 0px まで圧縮され、項目と操作の対応を判断できない
- 削るべき情報・操作: 各行で反復していた公開説明文と、音声・語義・生成・削除の常時表示
- 検証仮説・成功指標: 右レール併設時も一覧が主領域から横にはみ出さず、全項目が同じ x 座標と幅を持ち、見出し語領域が 240px 以上、直接操作が「開く」「その他」に限定される

## 3. 初見理解

- 何の画面か分かるか: 「Lexicon」「保存済みWordPack」の見出しを維持している
- 今どこか分かるか: 左ナビゲーションの Lexicon 選択状態と、表示切替の「リスト」選択状態を維持している
- 何ができるか分かるか: 青い「開く」を主操作として常時表示し、補助操作は可視ラベル「その他」と見出し語を含む accessible name を持つメニューへ集約した
- 最初の有意味な行動: 対象行の「開く」で内容を確認する
- 操作結果を予測できるか: 「例文を生成」「音声を再生」「語義を表示」「ゲスト公開にする」「削除」の対象と動作が分かるラベルをメニューに表示する
- 失敗時に戻れるか: メニューは Esc または外側クリックで閉じ、Esc の場合は起点ボタンへフォーカスを戻す。削除確認は既存挙動を維持する

## 5. アクセシビリティ確認

- キーボード: 行ごとにチェックボックス、「開く」、「その他の操作」へ到達できる。メニュー内は上下矢印、Home、End で移動できる
- フォーカス: 既存 `focus-visible` を維持し、メニューを Esc で閉じると起点へ戻す。表示切替時は開いたメニューを閉じる
- 名前・ラベル: リストに「保存済みWordPackのリスト」、各チェックボックスとその他ボタンに見出し語を含む accessible name を付与した
- 見出し・構造: 見た目だけの `div` 群から `ul` / `li` へ変更し、視覚順と DOM 順を1列で一致させた
- コントラスト: 既存テーマトークンを基準にし、主操作は既存の青系ボタン、危険操作は既存の赤系メニュー項目を維持した
- ターゲットサイズ: 「開く」「その他の操作」は最小 40px 高
- エラー・ステータス: 今回は変更せず、Issue #545 の P0 として継続する
- 自動検査: role / accessible name / focus / keyboard を Testing Library で検証し、Playwright + axe で違反 0 件
- 手動確認: 安全なモックを使った Playwright 描画とスクリーンショットを確認

## 6. 視覚階層

- 主操作: 「開く」を唯一の強い直接操作として右端に配置
- 情報優先度: 見出し語と生成状態を1段目、公開状態・更新日・学習状況を2段目、補助操作をメニューに配置
- グルーピング: 選択、項目情報、項目操作の3領域を同一行内で分離
- 余白・密度: 各行の反復説明と5個以上の直接ボタンを削り、走査に必要な余白を確保
- 読みやすさ: 長い見出し語を `overflow-wrap: anywhere` で省略せず表示
- 狭幅・文字拡大: 760px 以下では操作を項目情報の下へ送る。単列を維持し、主領域の横スクロールを発生させない

## 7. コピー

- 用語: 既存の「開く」「語義」「公開中」「非公開」を維持
- ボタン・リンク: 補助操作には「その他」を可視表示し、見出し語を含む accessible name も付けた。公開先は「ゲスト公開にする / ゲスト公開を解除」と明示した
- エラー文: 変更なし。Issue #545 で対応予定
- 空状態: 変更なし。Issue #545 で対応予定
- disabled: 語義一括表示中は「語義は一括表示中」と理由を title で示す
- トーン: ユーザーを責める表現を追加していない

## 8. 熟練者効率

- 主要反復タスク: 一覧を走査し、対象を開く
- 手数: 「開く」は1操作のまま。補助操作は1操作増えるが、誤操作と視覚走査負荷を下げる
- 再入力・再選択: 表示切替、並び順、絞り込みの既存保存状態を維持
- 近道: メニュー内の上下矢印、Home、End、Esc を追加
- 初心者向け説明の影響: 各行の公開説明を削り、熟練者の反復走査を妨げない。公開状態は短い「公開中 / 非公開」で残す
- 判定: Pass

## 10. 反証レビュー

- 実装を落とす観点で見つけた問題: 単に1列化するだけでは直接操作が多く縦書き化するため、補助操作をメニューへ移した。menu role だけ付けて矢印操作がない状態も不十分なため、移動キーとフォーカス復帰を追加した。自動レビューで「…」だけでは視認できないことと公開先が曖昧なことを検出し、可視ラベルとゲスト公開の明示を追加した
- P0候補: 読み込み失敗と真の空が混同される既存問題は残っており、Issue #545 として分離済み
- 証跡不足: 実ユーザーによる時間計測、Windows の文字描画、ブラウザ実ズーム操作は未実施
- 残リスク: Linux の visual regression は最新実装 head で成功した。Windows 実機とブラウザ UI の実ズーム操作は未確認で、検索条件の可視化と全件検索は Issue #164 / #115 に残る
