# PR #483 Google OAuth locale 移行 UI/UX レビュー

## 1. 概要

- 対象PR / 作業: PR #483 `@react-oauth/google` 0.13.5 更新に伴う `locale` 指定先の移行
- 変更した画面・コンポーネント: ログイン画面（表示内容は維持）、`AuthProvider`、`GoogleLoginCard`
- 判定: Pass
- P0件数: 0
- P1件数: 0
- P2件数: 0

変更前の `GoogleLogin` 直接指定から、ライブラリの現行契約である `GoogleOAuthProvider` 指定へ `locale="ja"` を移した。日本語表示、認証操作、状態表示、レイアウトは変更しない。

## 2. ユーザー価値

- 対象ユーザー: WordPack に Google アカウントでログインする日本語利用者
- 利用文脈: 初回アクセスまたはセッション失効後のログイン
- ユーザー目的: Google アカウントで認証し、辞書データと設定を同期する
- 支援するタスク: 日本語で認識できる Google ログイン操作を選び、認証を開始する
- このUIが助ける理解・判断・行動: ログイン方法、同期される対象、ゲスト閲覧という代替手段を理解して選べる
- このUIがなければ困る点: 認証の入口とゲスト閲覧の選択肢を判断できない
- 削るべき情報・操作: 今回の依存更新では追加なし
- 検証仮説・成功指標: 日本語ボタンと既存の accessible name が維持され、既存テストと build が成功すること

## 3. 初見理解

- 何の画面か分かるか: `WordPack にサインイン` の見出しで分かる
- 今どこか分かるか: ページ見出しとログインダイアログで分かる
- 何ができるか分かるか: Google ログインとゲスト閲覧の2経路が明示される
- 最初の有意味な行動: `Google でログイン` または `ゲスト閲覧モード` を選ぶ
- 操作結果を予測できるか: 説明文から辞書データと設定の同期、セッションクッキー保存を予測できる
- 失敗時に戻れるか: 既存のエラー表示と再試行導線を維持する

## 10. 反証レビュー

- 実装を落とす観点で見つけた問題: `locale` を削除するだけでは Google ボタンの日本語表示がブラウザ設定依存になる
- 対応: `GoogleOAuthProvider locale="ja"` へ移し、Provider mock の回帰テストで固定
- P0候補: なし
- 証跡不足: Google の実認証完了はローカル用設定のため未実施
- 残リスク: 外部 Google Identity Services の実アカウント認証は CI / ローカル証跡の対象外
